# دليل الباك اند — ProjexiOn

شرح مجلدات وملفات **الباك اند** في المشروع (Laravel — PHP). لا يشمل هذا الملف واجهة المستخدم (Blade/CSS/JS) إلا عند الربط مع المسارات أو الـ Controllers.

---

## 1. ما هو الباك اند في هذا المشروع؟

| البند | الوصف |
|-------|--------|
| **التقنية** | Laravel (PHP 8.2+) |
| **الدور** | المصادقة، إدارة المستخدمين والأدوار، المشاريع، المجموعات، التقارير، الإشعارات، التقييمات، التواصل مع خدمة Python لفحص التشابه |
| **قاعدة البيانات** | MySQL / MariaDB عبر Laravel Eloquent |
| **الصلاحيات** | Spatie Laravel Permission (أدوار وصلاحيات) |

المجلدات الأساسية للباك اند:

- `app/` — منطق التطبيق (Controllers, Models, Services, Middleware)
- `config/` — إعدادات التطبيق وقاعدة البيانات والصلاحيات
- `database/` — Migrations و Seeders و Factories
- `routes/` — تعريف المسارات (Web و API)
- `bootstrap/` — تحميل Laravel وتسجيل المسارات والـ Middleware
- `storage/` — ملفات مرفوعة، سجلات، cache (يُستخدم من الباك اند)
- `public/` — نقطة الدخول `index.php` فقط من ناحية الباك اند

---

## 2. مجلد `app/` — قلب الباك اند

يحتوي على كل المنطق البرمجي للتطبيق.

### 2.1 `app/Http/Controllers/`

**الوظيفة:** استقبال الطلبات من المسارات، تنفيذ المنطق، وإرجاع Response (عرض أو redirect أو JSON).

| الملف | الوظيفة |
|-------|---------|
| **Controller.php** | Controller أساسي يُورث منه الباقي (مشترك). |
| **Auth/AuthController.php** | تسجيل الدخول العادي (`/login`)، دخول الإدارة (`/admin`)، التسجيل، الخروج، الدخول السريع (`quick-login`). يتحقق من دور المستخدم عند دخول الإدارة. |
| **DashboardController.php** | توجيه المستخدم المسجل حسب دوره (طالب، مشرف، لجنة، إدارة، مدير نظام) إلى view لوحة التحكم المناسبة. |
| **ProjectController.php** | إدارة المشاريع: index, create, store, show, edit, update, destroy؛ موافقة/رفض، أرشفة، فحص تشابه، مكتبة المشاريع؛ إدارة الأقسام والملفات والموافقات على الأقسام. |
| **ProjectPhaseController.php** | أجزاء المشروع (Phases): index, create, store, show, edit, update, destroy؛ رفع ملفات، فحص تشابه للأجزاء/الملفات، تحميل الملفات. |
| **ProjectReportController.php** | تقارير المشاريع (المرحلية): index, store؛ عرض تقارير مشروع معين. |
| **GroupController.php** | المجموعات: CRUD، أعضاء، إضافة/إزالة طالب، مديرون، طلبات انضمام (قبول/رفض). |
| **ReportsController.php** | صفحة التقارير العامة: فلترة وبحث حسب اسم المشروع، نسبة التشابه، نسبة الإنجاز؛ عرض جدول المشاريع مع النتائج. |
| **EvaluationController.php** | تقييمات لجنة التقييم: إنشاء، عرض، تعديل، حذف؛ توليد PDF للتقييم. |
| **SupervisorEvaluationController.php** | تقييمات المشرف: إنشاء، عرض، تعديل، قائمة. |
| **NotificationController.php** | الإشعارات: صفحة index، قائمة (للاستخدام في الواجهة)، تعليم كمقروء، تعليم الكل كمقروء، عداد غير المقروءة، إرسال إشعار، تحميل مرفق، حذف. |
| **UserController.php** | إدارة المستخدمين (للإدارة فقط): index, show, edit, update, ban, unban, destroy. |
| **ProfileController.php** | الملف الشخصي: عرض نموذج التعديل وتحديث البيانات. |
| **SpecialtyController.php** | البحث عن التخصصات (لنماذج التسجيل وغيرها) — يُستخدم عبر AJAX. |
| **API/AISimilarityController.php** | واجهة API لفحص التشابه: استدعاء خدمة Python وإرجاع النتائج. |

### 2.2 `app/Models/`

**الوظيفة:** تمثيل جداول قاعدة البيانات والعلاقات بينها (Eloquent ORM).

| الملف | الجدول (تقريباً) | الوظيفة |
|-------|-------------------|---------|
| **User.php** | `users` | المستخدم: علاقات مع المشاريع، المجموعات، الإشعارات، الأدوار (Spatie). دوال مثل `hasRole`, `unreadNotifications`. |
| **Project.php** | `projects` | المشروع: مجموعة، أجزاء، أقسام، ملفات، تقارير، تقييمات، ملاحظات، فحوصات تشابه. دوال مساعدة للأرشفة والحالة. |
| **Group.php** | `groups` | المجموعة: مشرف، طلاب، مديرون، مشاريع، طلبات انضمام. |
| **ProjectPhase.php** | `project_phases` | جزء المشروع: أقسام، ملفات، نسب إنجاز، مواعيد تسليم. |
| **ProjectSection.php** | `project_sections` | قسم داخل جزء: ملفات، نسبة إنجاز، موافقة. |
| **ProjectFile.php** | `project_files` | ملف مرفق بمشروع أو قسم (PDF, DOCX...)؛ حقول تشابه. |
| **PhaseFile.php** | `phase_files` | ملف مرفق بجزء (Phase)؛ حقول تشابه واحتمال AI. |
| **ProjectReport.php** | `project_reports` | تقرير مرحلي للمشروع. |
| **ProjectNote.php** | `project_notes` | ملاحظة على مشروع. |
| **Evaluation.php** | `evaluations` | تقييم لجنة التقييم: درجات، تعليقات، نوع التقييم. |
| **SupervisorEvaluation.php** | `supervisor_evaluations` | تقييم المشرف. |
| **Notification.php** | `notifications` | إشعار: مستلم، عنوان، رسالة، قراءة، نوع، بيانات إضافية (مثل مرفق). |
| **SimilarityCheck.php** | `similarity_checks` | سجل نتيجة فحص تشابه (مشروع أو ملف). |
| **Specialty.php** | `specialties` | التخصص (للطلاب والمشاريع). |
| **Deadline.php** | `deadlines` | موعد تسليم. |
| **GroupJoinRequest.php** | `group_join_requests` | طلب انضمام طالب لمجموعة. |

### 2.3 `app/Services/`

**الوظيفة:** منطق أعمال معقد أو تواصل مع خدمات خارجية (مثل Python)، بعيداً عن الـ Controller.

| الملف | الوظيفة |
|-------|---------|
| **AISimilarityService.php** | إرسال طلبات إلى خدمة Python (فحص تشابه المشاريع/الملفات، اكتشاف المحتوى المولد بالذكاء الاصطناعي). قراءة `AI_SIMILARITY_URL` من الإعدادات. |
| **FileSimilarityService.php** | تجهيز البيانات واستدعاء خدمة التشابه للملفات؛ تنسيق النتائج. |
| **NotificationService.php** | إنشاء وإرسال الإشعارات (مثلاً عند موافقة/رفض مشروع، إشعار مخصص). |

### 2.4 `app/Http/Middleware/`

**الوظيفة:** تشغيل كود قبل/بعد وصول الطلب إلى الـ Controller (مثل التحقق من الصلاحيات أو الحظر).

| الملف | الوظيفة |
|-------|---------|
| **EnsureUserNotBanned.php** | التحقق من أن المستخدم غير محظور (`banned_at`). إن كان محظوراً يتم منع الوصول وتوجيهه مع رسالة. |

في `bootstrap/app.php` يتم تسجيل الـ Middleware التالية:

- `role` — التحقق من دور المستخدم (Spatie).
- `permission` — التحقق من صلاحية (Spatie).
- `role_or_permission` — دور أو صلاحية (Spatie).
- `ensure.not.banned` — الاستخدام الفعلي لـ `EnsureUserNotBanned`.

### 2.5 `app/Helpers/`

**الوظيفة:** دوال مساعدة عامة لا تتبع Model أو Controller.

| الملف | الوظيفة |
|-------|---------|
| **PdfHelper.php** | مساعدات لتوليد ملفات PDF (مثلاً تقارير التقييم). |

### 2.6 `app/Providers/`

**الوظيفة:** تسجيل خدمات في الحاوية (Service Container) وإعداد التطبيق عند البدء.

| الملف | الوظيفة |
|-------|---------|
| **AppServiceProvider.php** | مزود خدمة التطبيق العام (يمكن فيه تسجيل bindings أو إجراءات عند التحميل). |

## 3. مجلد `config/`

**الوظيفة:** إعدادات التطبيق وقاعدة البيانات والجلسات والصلاحيات وغيرها. القيم الحساسة تُقرأ من `.env`.

| الملف | الوظيفة |
|-------|---------|
| **app.php** | اسم التطبيق، البيئة، locale، timezone، URL، إعدادات عامة. |
| **auth.php** | إعدادات المصادقة: guards، providers (مثل استخدام جدول `users`). |
| **database.php** | اتصالات قواعد البيانات (MySQL وغيرها)؛ يُستخدم من `.env`: `DB_CONNECTION`, `DB_HOST`, `DB_DATABASE`, إلخ. |
| **permission.php** | إعدادات حزمة Spatie Laravel Permission (جداول الأدوار والصلاحيات، cache). |
| **session.php** | إعدادات الجلسة (driver، cookie، lifetime). |
| **filesystems.php** | أقراص التخزين (محلي، public، إلخ)؛ أين تُحفظ الملفات المرفوعة. |
| **cache.php** | إعدادات الـ cache. |
| **queue.php** | إعدادات قوائم الانتظار (إن وُجدت). |
| **mail.php** | إعدادات البريد (إن وُجد إرسال بريد). |
| **logging.php** | إعدادات السجلات (قنوات، مستويات). |
| **services.php** | إعدادات خدمات خارجية (يمكن أن يحتوي على عناوين خدمات مثل AI). |
| **dompdf.php** | إعدادات DomPDF لتوليد PDF. |

## 4. مجلد `database/`

**الوظيفة:** تعريف بنية قاعدة البيانات (Migrations)، البيانات الأولية (Seeders)، ونماذج بيانات للاختبار (Factories).

### 4.1 `database/migrations/`

كل ملف يصف إنشاء أو تعديل جدول (أو جداول). تُنفَّذ بالأمر: `php artisan migrate`.

| النوع / الاسم (تقريباً) | الوظيفة |
|--------------------------|---------|
| `*_create_users_table.php` | جدول المستخدمين. |
| `*_create_permission_tables.php` | جداول Spatie (أدوار، صلاحيات، ربطها بالمستخدمين). |
| `*_add_fields_to_users_table.php` | إضافة حقول إضافية للمستخدمين. |
| `*_create_groups_table.php` | جدول المجموعات. |
| `*_create_projects_table.php` | جدول المشاريع. |
| `*_create_project_phases_table.php` | أجزاء المشروع. |
| `*_create_project_sections_table.php` | أقسام داخل الجزء. |
| `*_create_project_files_table.php` | ملفات المشروع. |
| `*_create_phase_files_table.php` | ملفات الأجزاء. |
| `*_create_project_reports_table.php` | تقارير المشاريع. |
| `*_create_project_notes_table.php` | ملاحظات المشروع. |
| `*_create_evaluations_table.php` | تقييمات لجنة التقييم. |
| `*_create_supervisor_evaluations_table.php` | تقييمات المشرف. |
| `*_create_notifications_table.php` | الإشعارات. |
| `*_create_similarity_checks_table.php` | سجلات فحص التشابه. |
| `*_create_deadlines_table.php` | المواعيد. |
| `*_create_group_join_requests_table.php` | طلبات انضمام المجموعات. |

| `*_create_specialties_table.php` | التخصصات. |
| `*_add_*_to_*` | إضافة أعمدة (مثل `banned_at`, `archived_at`, `similarity_score`, `ai_probability`, إلخ). |
| `*_create_cache_table.php`, `*_create_jobs_table.php` | جداول Laravel للـ cache والـ jobs. |

### 4.2 `database/seeders/`

**الوظيفة:** إدخال بيانات أولية أو تجريبية. تُنفَّذ بالأمر: `php artisan db:seed`.

| الملف | الوظيفة |
|-------|---------|
| **DatabaseSeeder.php** | يشغّل بالترتيب: `RoleSeeder` → `SpecialtySeeder` → `QuickLoginSeeder`. |
| **RoleSeeder.php** | إنشاء الأدوار (admin, department_admin, supervisor, committee, student) والصلاحيات وربطها. |
| **SpecialtySeeder.php** | إدخال التخصصات الأولية. |
| **QuickLoginSeeder.php** | إنشاء مستخدمين تجريبيين (طالب، مشرف، لجنة، مدير نظام) مع كلمة مرور موحدة. |

### 4.3 `database/factories/`

**الوظيفة:** توليد بيانات وهمية للنماذج (مثلاً للاختبارات). مثال: `UserFactory.php` لإنشاء مستخدمين تجريبيين.

## 5. مجلد `routes/`

**الوظيفة:** ربط عناوين الطلبات (URL) والطرق (GET, POST, …) مع الـ Controllers أو الـ Closures.

### 5.1 `routes/web.php`

يحتوي على كل مسارات الويب الرئيسية:

- **عامة (بدون تسجيل):** `/` (الصفحة الرئيسية)، `/specialties/search`.
- **ضيف (guest):** `/login`, `/admin`, `/register`, `/quick-login/{role}`.
- **بعد تسجيل الدخول (auth + ensure.not.banned):**
  - لوحة التحكم: `/dashboard`.
  - التقارير: `/reports` (مع صلاحية دور).
  - الملف الشخصي: `/profile/edit`, `/profile`.
  - المستخدمون: `/users`, `/users/{user}`, تعديل، حظر، حذف (للإدارة).
  - المجموعات: مسارات `groups` (CRUD، أعضاء، طلبات انضمام).
  - المشاريع: مسارات `projects` (CRUD، موافقة، رفض، أرشفة، تشابه، مكتبة، أقسام، ملفات).
  - أجزاء المشروع: مسارات `project-phases` و `project-phases/{phase}`.
  - تقارير المشاريع: مسارات `project-reports`.
  - الإشعارات: `/notifications`, قراءة، حذف، إرسال، عداد غير مقروء.
  - التقييمات: مسارات `evaluations` و `supervisor-evaluations`.
  - تحميل ملفات الأجزاء وفحص التشابه للملفات.

يُحمَّل في `bootstrap/app.php` عبر `web: __DIR__.'/../routes/web.php'`.

### 5.2 `routes/api.php`

مسارات قد تُستخدم كـ API (مثلاً مع `auth:sanctum` أو `auth`):

- `/user` — بيانات المستخدم الحالي.
- `/ai/check-similarity` — فحص التشابه (AISimilarityController).
- `/ai/similarity-results/{project}` — نتائج تشابه مشروع.
- `/groups/find/{code}` — البحث عن مجموعة برمزها.

ملاحظة: في الإعداد الحالي في `bootstrap/app.php` يتم تحميل مسارات `web` و `commands` فقط؛ إذا أردت استخدام مسارات API يجب إضافتها في `withRouting` (مثلاً `api: __DIR__.'/../routes/api.php'`).

### 5.3 `routes/console.php`

أوامر الطرفية (Artisan) المجدولة أو المخصصة — جزء من تشغيل الباك اند.

## 6. مجلد `bootstrap/`

**الوظيفة:** تحميل إطار Laravel وتحديد المسارات والـ Middleware ومعالجة الأخطاء.

| الملف | الوظيفة |
|-------|---------|
| **app.php** | إنشاء التطبيق، تحميل مسارات `web` و `commands`، تسجيل Middleware (role, permission, ensure.not.banned)، معالجة الاستثناءات. |

## 7. مجلد `storage/`

**الوظيفة:** تخزين ملفات ينتجها الباك اند (ملفات مرفوعة، سجلات، cache، جلسات). لا يُخدم مباشرة للمتصفح إلا عبر ربط رمزي من `public/storage`.

| المسار (تقريباً) | الوظيفة |
|-------------------|---------|
| `storage/app/` | ملفات التطبيق (مثلاً الملفات المرفوعة من المستخدمين). |
| `storage/app/public/` | ملفات يُفترض أن تكون قابلة للوصول عبر الرابط (بعد `php artisan storage:link`). |
| `storage/logs/` | ملفات السجلات (Laravel log). |
| `storage/framework/cache/` | cache الملفات. |
| `storage/framework/sessions/` | بيانات الجلسة عند استخدام file driver. |

---

## 8. مجلد `public/`

من ناحية الباك اند:

| الملف | الوظيفة |
|-------|---------|
| **index.php** | نقطة الدخول الوحيدة لجميع الطلبات؛ يحمّل Laravel ويحوّل الطلب إلى المسارات والـ Controllers. |

---

## 9. تدفق الطلب في الباك اند (مختصر)

1. الطلب يصل إلى `public/index.php`.
2. `bootstrap/app.php` يُحمّل التطبيق ويُسجّل المسارات والـ Middleware.
3. `routes/web.php` يحدد أي Controller أو Closure يُستدعى حسب URL والـ Method.
4. الـ Middleware (مثل `auth`, `ensure.not.banned`, `role`) تُنفَّذ قبل الوصول إلى الـ Controller.
5. الـ Controller يستخدم الـ Models والـ Services لتنفيذ المنطق ويُرجع عرض (view) أو redirect أو JSON.
6. الـ Models تقرأ/تكتب من قاعدة البيانات حسب إعدادات `config/database.php` وملف `.env`.

## 10. ملخص المجلدات

| المجلد | الوظيفة في الباك اند |
|--------|------------------------|
| **app/** | Controllers، Models، Services، Middleware، Helpers، Providers — كل المنطق. |
| **config/** | إعدادات التطبيق، قاعدة البيانات، المصادقة، الصلاحيات، الملفات، الجلسات، إلخ. |
| **database/** | Migrations (بنية الجداول)، Seeders (بيانات أولية)، Factories (بيانات تجريبية). |
| **routes/** | ربط الـ URL بالـ Controllers (web و api و console). |
| **bootstrap/** | تحميل Laravel وتسجيل المسارات والـ Middleware. |
| **storage/** | تخزين الملفات المرفوعة، السجلات، الـ cache، الجلسات. |
| **public/** | نقطة الدخول `index.php` فقط. |

هذا الملف يقتصر على شرح الباك اند ومجلداته ووظيفة كل مجلد؛ للتوثيق العام والتثبيت راجع `README.md` و `دليل_المشروع.md`.
